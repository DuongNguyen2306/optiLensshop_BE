  OPTILENSSHOP — BACKEND: TÓM TẮT NGHIỆP VỤ VÀ PHÂN QUYỀN THEO ROLE
  (Mô tả bám sát code trong repo; cập nhật khi API thay đổi.)

1. TỔNG QUAN HỆ THỐNG
----------------------
- Backend Node.js/Express + MongoDB, phục vụ cửa hàng kính áp tròng / lens (sản
  phẩm, biến thể SKU, combo, giỏ hàng, đơn hàng, thanh toán, tồn kho, thống kê).
- Xác thực: JWT (Bearer). Hầu hết thao tác nhạy cảm yêu cầu đăng nhập và đúng role.
- Đăng ký công khai tạo tài khoản người dùng mặc định role "customer" (theo schema).


2. CÁC ROLE TRONG HỆ THỐNG
--------------------------
  customer     — Khách mua hàng.
  sales        — Nhân viên bán hàng: xác nhận đơn sau khi khách đặt.
  operations   — Vận hành/gia công: cập nhật trạng thái đơn (luồng cố định) và
                 thao tác API Ops cho đơn cần gia công lens.
  manager      — Quản lý cửa hàng: CRUD danh mục dữ liệu, nhân viên (sales/ops),
                 xem thống kê, kho; xem toàn bộ đơn; can thiệp Ops.
  admin        — Quản trị cao nhất: mọi quyền của manager + quản lý manager +
                 một số API thống kê chỉ dành cho admin.

Lưu ý tạo tài khoản nội bộ (theo user.service):
  - Manager/Admin tạo nhân viên với role chỉ được "sales" hoặc "operations".
  - Chỉ Admin tạo/sửa/xóa tài khoản "manager".


3. LOẠI ĐƠN HÀNG (order_type) VÀ Ý NGHĨA NGHIỆP VỤ
---------------------------------------------------
  stock         — Hàng có sẵn trong kho (biến thể còn đủ tồn). Luồng trạng thái
                  ngắn hơn: không qua manufacturing/received.
  prescription  — Đơn theo đơn kính (có ảnh/chỉ định bác sĩ, v.v. tùy payload).
                  Luồng có bước MANUFACTURING (gia công).
  pre_order     — Đặt trước khi biến thể hết hàng hoặc đặt preorder trực tiếp.
                  Luồng có bước RECEIVED (hàng về) trước khi đóng gói.

Cờ nghiệp vụ liên quan:
  - requires_fabrication: đồng bộ true khi order_type = prescription (pre-save).
  - Đơn hiển thị trên API Ops (/api/ops/orders) phải là đơn "gia công" (requires_fabrication
    hoặc lens_worksheet.is_custom_made) và đang ở các trạng thái Ops cho phép.


4. TRẠNG THÁI ĐƠN HÀNG VÀ LUỒNG CHUYỂN (tóm tắt)
--------------------------------------------------
Các trạng thái: pending → confirmed → processing → (manufacturing HOẶC received,
  tùy loại đơn) → packed → shipped → delivered → completed; hoặc cancelled;
  sau giao có thể return_requested → returned/refunded.

Khác biệt theo loại (ORDER_TRANSITIONS trong code):
  - stock:      processing → packed hoặc shipped (tùy cấu hình chuyển), không có
                manufacturing/received.
  - prescription: processing → manufacturing → packed → shipped → ...
  - pre_order:    processing → received → packed → shipped → ...

Ai được đổi trạng thái:
  - Sales: POST /orders/:id/confirm — chuyển sang confirmed (theo luồng hợp lệ).
  - Operations: PUT /orders/:id/status — chỉ cho các trạng thái thuộc nhóm
    "OPS_ONLY" (processing, manufacturing, received, packed, shipped, delivered);
    nếu role khác gọi sẽ bị từ chối ở tầng service.
  - Operations / Manager / Admin: các route PATCH dưới /api/ops/orders — điều phối
    bắt đầu xử lý, hoàn tất gia công, giao vận chuyển, đánh dấu đã giao (chi tiết mục 6).

Thanh toán COD: khi đơn chuyển delivered, hệ thống có thể tự cập nhật payment COD
sang paid (trong order.service / ops-order.service).


5. CÁC NHÓM NGHIỆP VỤ CHÍNH (THEO MODULE API)
---------------------------------------------
  Auth (/auth)
    Đăng ký, đăng nhập, xác thực email, quên/đổi mật khẩu — không phân role trên route;
    sau đăng nhập, quyền theo JWT.

  Người dùng (/users)
    Profile và địa chỉ của chính user đang đăng nhập (mọi role đều có thể gọi nếu
    dùng tài khoản đó).

  Catalog công khai (không cần role đặc biệt)
    GET /products, GET /products/:slug, GET /products/:id/variants
    GET /categories, GET /combos, GET /combos/:slug
    GET /brands, GET /models
    GET /variants (theo type)

  Quản trị danh mục & sản phẩm (manager + admin)
    POST/PUT/DELETE /categories, /brands, /models
    POST/PUT/PATCH/DELETE /products và biến thể
    POST/PUT/DELETE /combos

  Giỏ hàng (/cart)
    Mọi user đã authenticate — thực tế dùng cho khách mua; không khóa role ở route.

  Đơn hàng (/orders)
    - Khách: danh sách đơn của mình, checkout, preorder-now, hủy đơn (customer).
    - Sales/manager/operations/admin: GET /orders/all — danh sách toàn shop.
    - Sales: xác nhận đơn.
    - Operations: PUT /orders/:id/status — cập nhật trạng thái nhóm Ops.
    - GET /orders/:id: chỉ trả về chi tiết nếu order.user_id trùng user đang đăng
      nhập; nhân viên không xem được đơn khách qua route này (chỉ có danh sách /all
      trong code hiện tại).

  Ops đơn gia công (/api/ops/orders)
    operations, manager, admin: danh sách đơn Ops, start-processing, fulfill,
    start-shipping, delivered — áp dụng cho đơn đủ điều kiện "fabrication" và trạng
    thái hợp lệ (xem ops-order.service).

  Thanh toán
    /momo, /vnpay: tạo thanh toán / IPN / return — route không gắn authorize trong code;
    frontend/backend tích hợp cần bảo vệ logic nghiệp vụ (orderId, amount).
    /payment/success, /payment/fail: cần authenticate.

  Kho (/inventory) — manager + admin
    Phiếu nhập kho (receipts), xác nhận phiếu, sổ cái (ledger).

  Thống kê (/statistics) — manager + admin
    overview, timeseries, top-products, inventory-alerts, funnel.
    Riêng GET /statistics/admin — chỉ admin.

  Quản lý nhân sự (/management)
    manager + admin: CRUD staff (sales/operations).
    admin only: CRUD managers.


6. BẢNG "ROLE LÀM ĐƯỢC GÌ" (TÓM TẮT THAO TÁC)
-----------------------------------------------
  CUSTOMER
    - Duy trì giỏ hàng, đặt hàng (checkout / preorder-now), xem/hủy đơn của mình.
    - Xem chi tiết đơn của mình (GET /orders/:id).
    - Auth, profile, địa chỉ.
    - Xem catalog công khai.

  SALES
    - Xem tất cả đơn (GET /orders/all).
    - Xác nhận đơn: POST /orders/:id/confirm.
    - Không được route Ops PATCH; không được PUT /orders/:id/status (operations only).

  OPERATIONS
    - Xem tất cả đơn (GET /orders/all).
    - Cập nhật trạng thái đơn qua PUT /orders/:id/status (nhóm trạng thái Ops).
    - Toàn bộ /api/ops/orders (cùng manager, admin).

  MANAGER
    - Toàn quyền CRUD dữ liệu: sản phẩm, biến thể, combo, category, brand, model.
    - Kho: receipts, ledger.
    - Thống kê (trừ endpoint /statistics/admin).
    - Quản lý staff (sales/operations).
    - Xem đơn toàn shop; can thiệp Ops giống operations trên /api/ops/orders.

  ADMIN
    - Tất cả quyền của manager +
    - Quản lý manager (CRUD /management/managers).
    - GET /statistics/admin.


7. GHI CHÚ TRIỂN KHAI
---------------------
- Base URL và cổng theo biến môi trường (mặc định app thường PORT 3000).
- Tài liệu OpenAPI: /api-docs và /openapi.json.
- Một số endpoint thanh toán không có middleware role: cần kiểm soát ở tầng triển
  khai (reverse proxy, secret, hoặc bổ sung auth nếu yêu cầu bảo mật cao hơn).
